# 배포 가이드: industry.truthofmarket.com

이 가이드는 풀스택 Industry Portal 애플리케이션을 사용자님의 서버에 서브도메인 **`industry.truthofmarket.com`**으로 배포하는 방법을 단계별로 설명합니다.

---

## 1. 도메인 및 DNS 설정

도메인 관리 사이트(예: 가비아, 후이즈 등)의 DNS 설정 페이지에서 `truthofmarket.com`에 대해 아래와 같이 DNS 레코드를 추가합니다.

| 레코드 타입 | 호스트 (서브도메인) | 값 (대상 IP) | TTL |
|------|------------------|-------------------|-----|
| **A** | `industry` | `사용자님의_서버_공인_IP` | 자동 (또는 기본값) |

---

## 2. 프론트엔드 빌드 (Vite)

배포하기 전에 프론트엔드를 빌드하여 정적 파일을 생성해야 합니다.

1. 프론트엔드 디렉터리로 이동합니다:
   ```bash
   cd D:\Industry\InvestmentPortal\frontend
   ```
2. 프로젝트를 빌드합니다:
   ```bash
   npm run build
   ```
   빌드가 완료되면 `dist` 디렉터리가 생성됩니다. 이 정적 폴더의 파일들을 Nginx가 웹에 서빙하게 됩니다.

---

## 3. 백엔드 배포 및 설정

서버의 특정 경로(예: `/var/www/industry/backend`)로 백엔드 파일들을 업로드합니다.

1. **파이썬 가상환경 구축 및 패키지 설치**:
   ```bash
   cd /var/www/industry/backend
   python3 -m venv venv
   source venv/bin/activate
   pip install --upgrade pip
   pip install -r requirements.txt
   ```
2. **환경 변수 파일 생성**:
   `/var/www/industry/backend/.env` 파일을 생성하고 아래 내용을 입력합니다:
   ```env
   DEEPSEEK_API_KEY=YOUR_DEEPSEEK_API_KEY
   ```
3. **데이터베이스 초기화 및 시딩**:
   서버 DB에 미리 준비된 한국어 데이터와 프로필을 입력합니다:
   ```bash
   python populate_korean_content.py
   ```
4. **Systemd 서비스 파일 등록 (백그라운드 유지)**:
   FastAPI 백엔드가 서버 재부팅 시에도 자동으로 백그라운드에서 상시 실행되도록 `/etc/systemd/system/industry-backend.service` 파일을 생성합니다:
   ```ini
   [Unit]
   Description=FastAPI Backend for Industry Portal
   After=network.target

   [Service]
   User=www-data
   WorkingDirectory=/var/www/industry/backend
   ExecStart=/var/www/industry/backend/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000
   Restart=always
   EnvironmentFile=/var/www/industry/backend/.env

   [Install]
   WantedBy=multi-user.target
   ```
5. **백엔드 서비스 시작**:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable industry-backend
   sudo systemctl start industry-backend
   ```

---

## 4. Nginx 리버스 프록시 설정

Nginx 설정 파일 `/etc/nginx/sites-available/industry.truthofmarket.com`을 생성하고 아래 설정을 입력합니다:

```nginx
server {
    listen 80;
    server_name industry.truthofmarket.com;

    # 프론트엔드 정적 파일 서빙
    root /var/www/industry/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 백엔드 API 프록시 우회
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # PDF 정적 파일 프록시 우회 (backend/산업자료 폴더 내 PDF 서빙)
    location /pdfs/ {
        proxy_pass http://127.0.0.1:8000/pdfs/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
    }
}
```

설정을 활성화하고 Nginx를 재시작합니다:
```bash
sudo ln -s /etc/nginx/sites-available/industry.truthofmarket.com /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 5. SSL 보안 인증서 적용 (HTTPS)

Certbot을 사용해 `industry.truthofmarket.com` 서브도메인에 무료 SSL 인증서(Let's Encrypt)를 적용합니다:

```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d industry.truthofmarket.com
```

인증서 발급 진행 시, HTTP 트래픽을 HTTPS로 자동 리다이렉트(Redirect)하는 옵션을 선택해 주면 Certbot이 Nginx 설정을 자동으로 수정해 줍니다.
